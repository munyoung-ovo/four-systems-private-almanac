# 启动与菜单

不要求固定唤醒词。用户已有明确问题时直接进入对应模块，不扫描无关目录、不检查全部依赖、不展示菜单。

只有用户想浏览功能、仅说“开始/载入”，或意图不清时，才读取当前档案状态并显示：

```text
━━━━━━━━━━━━━━━━━━━━━━━
当前：[姓名或尚未建档]
━━━━━━━━━━━━━━━━━━━━━━━
① 今日与择日
② 日历/图文
③ 命盘解读
④ 合盘
⑤ 命盘册

也可以直接问具体问题。
━━━━━━━━━━━━━━━━━━━━━━━
```

## 尚未建档

需要命盘功能时，请用户自然发送：姓名、性别、出生日期、出生时间、出生地，关系备注可选。资料完整且无高风险边界时可软确认后直接建档；历法不明、时辰不精确或材料冲突时按 `reference/build_profile.md` 强确认。

用户只浏览菜单时不强迫建档。日历、命盘、合盘等真正需要档案的功能再引导建档。

## 路由

| 用户意图 | 模块 |
| --- | --- |
| 出生信息、建档、改盘、材料核对 | `reference/build_profile.md` |
| 今天、某日、适不适合、挑日期 | `reference/module_1_today.md` |
| ICS、HTML、行动图、关系说明书 | `reference/module_2_calendar.md` |
| 命盘、事业、感情、财运、贵人、健康 | `reference/module_3_chart.md` |
| 合盘、关系比较、共同好日子 | `reference/module_4_heban.md` |
| 切换、新增、删除、档案排名 | `reference/module_5_profiles.md` |

排序和权重入口仅在用户主动询问系统优先级时展示。意图仍不清时只问一个最短问题，例如：“你想看某一天，还是看整张命盘？”
