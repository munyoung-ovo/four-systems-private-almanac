# 启动与菜单

不要求固定唤醒词。把“载入 skill”“载入这个 skill”“载入黄道吉日”“使用黄道吉日”“打开黄道吉日”都视为成功启动，不要求用户改用斜杠命令或重复输入。用户已有明确问题时直接进入对应模块，不扫描无关目录、不检查全部依赖、不展示菜单。

只有用户只表达启动、想浏览功能，或意图确实不清时，才读取当前档案状态并显示：

## 入口判定

- 只说“载入”“载入 skill”“载入黄道吉日”“开始”“打开”：先检查是否有当前档案；有档案立即显示主菜单，没有档案则直接进入首次资料填写引导。不得回复“无法载入”“请使用固定命令”或要求用户重述。
- 启动语与具体问题出现在同一句，例如“载入黄道吉日，帮我看今天适不适合签约”：把启动语当作前缀，直接处理后面的具体问题，不显示菜单。
- 用户直接问领域问题或自然说法，例如“帮我看看工作”“我和他合不合”“下周哪天面试好”：按含义路由到对应模块；缺少会改变结论的资料时，只追问最关键的一项。
- 用户只说“我想问个问题”“帮我看看”而没有领域：用一句短问承接，例如“想看今天与择日，还是命盘与关系？”；不要假装 skill 没有启动。

```text
━━━━━━━━━━━━━━━━━━━━━━━
当前：[姓名]（[关系备注或未标注]）
━━━━━━━━━━━━━━━━━━━━━━━
① 今日与择日  看今天、某件事或挑日期
② 日历/图文    生成手机日历、行动图或关系说明书
③ 命盘解读    看整体、事业、感情、财运与具体困惑
④ 合盘        看两个人的相处与共同窗口
⑤ 命盘册      切换、新增或管理档案

直接问也行，例如：“下周哪天面试好”“我想看事业”“我和他合不合”。
━━━━━━━━━━━━━━━━━━━━━━━
```

## 首次资料填写

当前没有档案时，不能只在菜单里写“尚未建档”。用户只说启动、或提出需要个人化结果的问题时，直接邀请其按下面格式发送资料；不要先要求选“命盘解读”或再问一次要不要建档：

姓名：
性别：
出生日期：YYYY-MM-DD
出生时间：HH:MM（不确定可写大概时段）
出生地：城市/区县

用户在启动语后已经附带全部资料时，直接进入建档流程。只给出部分资料时，只追问缺失的关键项，不重新贴整张表。已经上传或放入 materials 文件夹的星盘、紫微、印占 PDF 或截图，也可以直接说“用这个资料建档/核盘”；先列出可选文件，再按建档规则识别与复审。用户明确说“先看功能”“先浏览菜单”时，才显示主菜单而不强迫填写。

需要命盘功能时，按上述格式收集姓名、性别、出生日期、出生时间、出生地，关系备注可选。资料完整且无高风险边界时可软确认后直接建档；历法不明、时辰不精确或材料冲突时按建档规则强确认。

首次建档完成后，立即展示简明命盘快照和当前最相关的下一步，不再把用户送回冷菜单。已经有档案的用户仍可从主菜单进入日历、命盘、合盘等功能。
