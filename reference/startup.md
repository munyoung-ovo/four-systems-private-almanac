# 启动与菜单

启动可以来自显式“载入黄道吉日”，也可以来自自然语言意图。不要要求用户必须说固定口令。

启动后先看两件事：

1. `profiles/` 里是否已有命盘档案。
2. `materials/` 里是否已有辅助资料。

## 已有档案

如果用户没有明确提出具体事项，直接显示主菜单，不解释分数、流程或内部规则。若用户已经问了事业、感情、择日、合盘、材料核盘等具体事项，直接进入对应模块。

```text
━━━━━━━━━━━━━━━━━━━━━━━
当前：[姓名]（[关系备注或未标注]）
━━━━━━━━━━━━━━━━━━━━━━━
① 今日算命
② 生成日历/图文 (ICS/HTML)
③ 深度命盘解读
④ 合盘
⑤ 命盘册

直接问也行 → “11月雅思能过吗”“我的对象在哪”
━━━━━━━━━━━━━━━━━━━━━━━
```

用户问四套系统的优先级时，再给出排序入口：

```text
a. 默认专业排序
b. 自定义排序：1八字 2紫微 3西占 4印占

例如：b4213 = 印占 > 紫微 > 八字 > 西占
```

## 没有档案

引导用户先建档，语言要像真人助手，不要像表单系统。

```text
━━━━━━━━━━━━━━━━━━━━━━━
黄道吉日 · 私人通勤
━━━━━━━━━━━━━━━━━━━━━━━
第一次使用，需要先建立一个命盘。

请直接发：
姓名 性别 出生日期 出生时间 出生地 [关系备注可选]

如果你有星盘、印占 PDF 等辅助资料，
可以先放进 materials/ 文件夹，然后直接输入“a”。
最好也一起发出生信息，方便我帮你核对。
━━━━━━━━━━━━━━━━━━━━━━━
```

如果 `materials/` 里已经有文件，列出文件名，问用户是用文件辅助建档，还是手动输入出生信息。文件只辅助识别出生资料，不替代本地排盘。

## 自然语言路由

| 用户说的内容 | 去哪里 |
| --- | --- |
| 直接发出生信息，如“1997年11月14日14:30，女，甘肃张掖” | `reference/build_profile.md` |
| 建档、排盘、帮这个人建命盘、星盘 PDF/紫微截图核对 | `reference/build_profile.md` |
| 今天、明天、本周、运势、适不适合 | `reference/module_1_today.md` |
| 哪天好、挑日子、签约、考试、搬家 | `reference/module_1_today.md` |
| 生成日历、导出、ICS、HTML、图文页、行动地图、双人关系地图 | `reference/module_2_calendar.md` |
| 命盘、命格、事业、感情、财运、贵人、健康 | `reference/module_3_chart.md` |
| 合盘、我和某人合不合、关系怎么样 | `reference/module_4_heban.md` |
| 命盘册、切换、新增、删除、今天谁最旺 | `reference/module_5_profiles.md` |

意图不清时，只问一句：

```text
你想看今天的状态，还是想让我帮你挑个好日子？
```

如果用户已经明确给出事项，不要先展示主菜单再让用户选；直接进入对应模块。主菜单只在用户无明确意图、想浏览功能，或模块结束后返回时显示。排序入口只在用户问系统优先级、排序、权重或自定义偏好时显示。
