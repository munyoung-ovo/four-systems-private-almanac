# Module 4 · 合盘

触发：用户选 ④，或说合盘、我和某人合不合、我们关系怎么样。

需要输出文案结构时按 `prompts/heban.md`。

## 确定对象

先看命盘册里有没有可选对象。有的话列出来；没有的话，请用户输入对方出生信息。

新人路径：

1. 识别姓名、性别、出生日期时间、出生地。
2. 展示识别结果，让用户确认。
3. 询问是否保存到命盘册。
4. 询问关系类型：情侣、家人、亲子、合伙人、朋友。

## 关系判断

合盘先看三件事：

- 两个人的关系底色：吸引、互补、消耗、稳定性。
- 今年关系气候：适合靠近、修复、定局，还是先观察。
- 未来 90 天共同窗口：有没有两个人都比较顺的日子。

## 输出

默认输出简略版：

- 双人重点窗口置顶。
- 今年关系气候。
- 一句关系钩子。
- 下一步入口：展开详细 / 找共同好日子 / 回主菜单。

用户要求详细版时，再写关系底色、今年完整气候、双方各自需要注意什么。

## 具体问题

用户问具体事情时先确认：

```text
你们在考虑[用户说的事]。
这是已经决定要一起做，还是还在评估合不合适？
```

然后分别看两个人是否同向、谁更急、谁更稳，以及何时出手更合适。

## 找共同好日子

先问事项和时间范围。若没有共同高分日，给双方各自较好的日期，再给一个折中日期。
