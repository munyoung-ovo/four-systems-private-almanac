# 合盘

触发：合盘、关系比较、两个人是否适合相处或共同择日。

## 准备

优先从命盘册选择两人。缺少对象时只补建对方档案；确认姓名、性别、出生日期时间、地点和关系类型。资料清晰时直接建档，存在历法、时辰或材料冲突时先确认。

## 执行

1. 分别读取双方档案和审计，不因用户对关系的描述改动任何一方盘面。
2. 默认比较关系底色、互补、摩擦、当前阶段和未来共同窗口。
3. 具体问题只分析相关领域；共同择日补充事项和日期范围后批量计算双方结果。
4. 默认按 `prompts/heban.md` 给标准详细答；用户明确要求详细版或选择 `[详细版/1]` 时再写文件。
5. 用 `engines.ui_state.set_chart_topic(...)` 保存关系类型、双方代号和模块名。用户回复 `1`、`详细版`、`展开`、`深度解读` 时，只在上一轮已给出详细版入口的上下文中生成该关系的详细版，不误路由到个人命盘。

某一方时辰未知时，保留不依赖时辰的关系证据并降低对应结论强度。没有可靠共同窗口时给扩大范围、双方分别择日或折中日期，不硬凑好日子。

不预测分手、离婚、背叛或必然结果。关系建议必须具体到行为、边界和沟通时机。标准详细答结尾默认把 `[详细版/1]` 放在第一个入口。
