# 占断与验证协议

## 三类结论

先判断当前证据能回答哪一层，不能跨层推断：

| 层次 | 回答什么 | 所需证据 |
| --- | --- | --- |
| 本命 | 长期惯性、需求和承受方式 | 不随日期变化的本命结构 |
| 阶段 | 当前几年或数月的主题、资源和压力 | 已刷新到目标日期的大运、流限或行运 |
| 事件/日期 | 某件事在某日如何推进 | 明确事项、日期和日/周级信号 |

本命倾向不能直接预测具体事件；阶段顺利不能证明某一天适合定局；单日压力也不能推翻长期方向。

## 主判断结构

每个主判断在内部必须包含：

1. `结论`：直接回答，不先复述用户。
2. `表现`：最多两个可观察场景，具体到行为、节奏或事件形式。
3. `时间`：只使用程序提供的目标日期、阶段或窗口；没有时间证据就不写应期。
4. `依据`：至少一条真实字段；强弱、吉凶和时间结论优先两套独立系统。
5. `反证`：哪条冲突、低置信或现实条件会让结论不成立。
6. `动作`：与结论强度匹配的一步；证据弱时只建议可逆动作。

普通短答不必显示六个标题，但内容不能缺失决定结论所需的部分。

## 置信语言

- `confidence >= 0.75` 且至少两套可靠系统同向：可以写“更可能、主要表现为”。
- `0.55-0.75` 或仅一套强信号：写“倾向、值得观察”，并给验证点。
- `< 0.55`、资料降级或同维度冲突：不下强结论，说明缺口并建议复核或小步试探。

票数不是置信度。四套低质量信号不能写成强判断，一套高质量反证也不能被聊天背景抹掉。

专题综合的 `grade` 决定表达上限：`strong` 可作主结论，`moderate` 必须带条件，`limited` 只能写单系统观察，`insufficient` 不输出方向性判断。

## 具体与可验证

- 不写“有时外向、有时内向”“渴望被理解”等无法证伪的句子。
- 把抽象词翻译成行为：例如“关系压力”要说明是承诺节奏、沟通延迟还是边界摩擦。
- 不从盘面结构推导姓名、长相、精确职业、具体金额、疾病、死亡或必然结果。
- 如果程序只给领域背景，就写领域背景；不要补成具体事件。

## 不符合时的校正顺序

用户说“不像/不准”时，不立刻换一种解释迎合。依次检查：

1. 出生时间、地点、时区、历法和材料归属。
2. 结论是否跨越了本命、阶段、事件三个层次。
3. 目标日期是否刷新，系统是否处于降级或边界状态。
4. 原结论的来源字段和反证是否完整。

资料无误仍不符合时，保留为未命中反馈，不把用户经历反写进当前盘面。只有积累到可回溯样本后才调整规则置信度。
