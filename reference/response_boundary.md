# 盘面与对话边界

## 结论锁定

先只使用档案、当前日期、事项和本地计算结果形成盘面结论，再读取有限对话信息安排建议。对话不得补造、修改或推翻盘面事实、系统权重、吉凶倾向、时间窗口及冲突类型。

每个核心判断必须能追溯到实际字段。涉及强弱、时间或吉凶时，优先采用两套以上独立系统的同向证据；只有一套弱信号时写成补充观察。系统冲突必须保留，并说明冲突来自时间、领域、层次还是强度。

## 信息权限

| 信息 | 可否影响盘面 | 用途 |
| --- | --- | --- |
| 出生日期、时间、地点、时区、历法及核验材料 | 可以 | 重排或调整精度 |
| 当前事项、目标日期、客观限制 | 只影响所用规则 | 选题、择日范围和行动建议 |
| 经历、情绪、担忧、自我评价 | 不可以 | 语气、举例和建议落地 |
| 明确保存的系统偏好 | 可以 | 仅通过 `meta.fusion_system_order/weights` 生效 |

## 不变性

同一档案、日期和问题，即使聊天背景相反，系统权重、投票、综合分、冲突类型、核心结论和时间范围必须一致。只允许建议措辞、安抚方式和追问不同。

用户反馈用于核验和积累结果记录，不得因一次“准/不准”即时改写当前盘面。需要调整系统置信度时，只使用明确保存、可回溯的校准数据。
