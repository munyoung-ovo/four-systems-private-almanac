# Module 2 · 日历与图文

触发：用户选 ②，或说生成日历、导出 ICS、导入手机日历、生成 HTML、图文页、行动地图、双人关系地图。

## 一级拦截菜单

用户在主菜单输入 `2` 时，绝对不要直接计算、预览或生成文件。先显示二级菜单，等待用户二次输入：

```text
请选择您要生成的格式：
1. ICS 手机日历 (导入系统日历)
2. 运势行动图 (未来30天节奏规划)
3. 关系说明书 (合盘底色 + 30天相处天气)
```

只有用户选择二级菜单后，才进入对应分支。

## 1. ICS 手机日历

用户选 `1` 时，才运行日历逻辑。

聊天里只预览未来 30 天内最关键的 3 到 4 条。预览后再生成 30 天 `.ics` 文件。

预览时突出：

- 不刷屏，只显示最值得看的 3 到 4 天。
- 同时照顾趋吉和避凶：尽量同时包含好日子和需要避开的日子。
- 每天只说适合什么、不适合什么，不展示分数和内部判断。
- 说明生成日历后会覆盖未来 30 天。

ICS 内部覆盖未来 30 天每日双轨；聊天预览不等于完整导出清单。

日历事件采用双轨制：

- 吉：当天最适合做的事。
- 凶：当天最不适合做的事。

手机日历标题保持极短，只写类似：

```text
宜签约.出门
忌开业.装修
```

DESCRIPTION 只允许白话结果，不写推导过程、八字术语、择日术语或冒号后的理由。宜轨和忌轨必须分开：

```text
【今日宜】祈福、出行、签约
```

```text
【今日忌】搬家、装修
```

禁止生成 `VALARM`。如果旧文件已经存在，按当前文件策略覆盖或提示用户确认。

生成后返回 `.ics` 路径，并给简短导入指引：

- 电脑：直接打开 `.ics`。
- iPhone：发到手机后点附件，选择添加全部日历事件。
- 安卓 / Google 日历：用 Google 日历打开，或在设置里导入。

## 2. 运势行动图

用户选 `2` 时，直接调用 HTML 渲染，不运行 ICS 预览。

默认从今天起算未来 30 天；若用户指定“下个月 / 2026-07”，从该月 1 日起算未来 30 天。

使用：

```python
from engines.render import render_treasure_map, save_render

html = render_treasure_map(profile, "YYYY-MM")
path = save_render(html, "action_map_[name]_[YYYY-MM].html", out_dir="outputs")
```

生成后只返回文件路径和自然入口，不展示源码。

## 3. 关系说明书

用户选 `3` 时，直接调用关系 HTML 渲染，不运行 ICS 预览。

先联动命盘册，让用户选择两个人：

```text
请选择要生成关系说明书的两个人：
1. [档案A]（[关系备注]）
2. [档案B]（[关系备注]）
3. 新增一个人
```

如果命盘册不足两人，先进入新增档案流程。新增完成后回到这里，再让用户选择两个人。

姓名策略：

- 默认显示用户选择的两个人姓名，更有代入感。
- 如果用户说“匿名 / 不显示姓名 / 用代号”，才改成“对象A / 对象B”或用户指定代号。
- 文件名不要带真实姓名，使用 `relationship_map_[timestamp].html`。

使用：

```python
from engines.render import render_relationship_map, save_render

html = render_relationship_map(profile_a, profile_b, days_ahead=30, show_names=True)
path = save_render(html, "relationship_map_[timestamp].html", out_dir="outputs")
```

生成后只返回文件路径和自然入口，不展示源码。

## 结束入口

结束时给自然入口：

```text
继续生成：ICS 日历 / 运势行动图 / 关系说明书
或回主菜单
```
