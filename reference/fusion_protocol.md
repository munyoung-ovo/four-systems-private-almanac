# 四系统融合

融合的目标是从八字、紫微、西占和印占中提炼一个可追溯、有限定、能执行的结论，而不是简单投票或罗列术语。

## 分工

| 系统 | 主要尺度 | 常用价值 |
| --- | --- | --- |
| 八字 | 本命、阶段、日课 | 底层需求、承受方式和阶段顺逆 |
| 紫微 | 宫位、年/月层次 | 问题所在领域和阶段主题 |
| 印占 | 阶段、日/时节律 | 启动、等待、收尾和择时 |
| 西占 | 天/周触发 | 近期外部变化、心理与事件触发 |

分工不是固定优先级。先看当前问题属于哪个领域和时间尺度，再看字段精度、信号强度和置信度。

## 内部综合

1. 把每套可用系统的相关信号翻译为同一问题下的短命题。
2. 删除不覆盖当前领域、时间尺度不匹配或已经降级的弱信号。
3. 多系统同向：作为主结论；两套可靠系统同向：给出领域或时间限定；单套信号：仅作补充。
4. 有冲突时区分时间、领域、层次和强度。真对立则降低确定性，建议可逆动作。
5. 形成“结论、依据、边界、动作”，不展示内部推理过程。

`weighted_score` 用于可靠方向和排序，`resonance_strength` 仅表示同向票数。用户对某套系统的偏好只有明确保存到档案设置后才能改变权重；普通聊天和一次反馈不能调权。

## 用户表达

不要说“四系统 3/4 支持”，也不要逐套报术语。可写：

```text
大方向可以推进，但今天更适合谈条件、补资料，不适合签字付款。
```

普通回答按以下顺序：

```text
结论：现在适合什么。
依据：最关键的盘面共识。
边界：冲突、精度或适用范围。
动作：现在可执行的一步。
```
