# 四系统断法协议

本文件约束从排盘字段到用户结论的推导。程序字段是事实，规则负责限定可说到哪一层；任何提示词不得越过字段精度补造事件。

## 通用证据顺序

1. 先确认出生时间、地点、历法、目标日期和审计状态。
2. 先读本命结构，再读当前阶段，最后读具体日期触发。三层不能互相替代。
3. 每套系统先在内部形成结论，保留支持、反证、适用领域和时间尺度，再参与四系统融合。
4. 主要结论优先采用两套以上独立且匹配问题尺度的证据。单套强信号只限定在该系统能覆盖的范围内。
5. `topic_signals.components` 用于检查结论由哪些真实层次组成；不得只读最终方向。

## 八字

主证据顺序：格局与月令 → 日主旺衰及置信度 → 调候 → 大运 → 流年。十神名称本身不是吉凶，必须结合日主承受能力、喜忌可信度和所问领域。

- `special_pattern` 非空、`strength_near_boundary=true` 或 `strength_confidence<0.6`：普通喜忌不得下强结论。
- 大运代表阶段主轴，流年代表该阶段中的年度触发。二者同向才可增强时间判断；反向时写成“背景与当年表现不同”。
- 感情不能只凭财官出现；事业不能只凭官杀印食；财运不能只凭财星。必须同时说明日主能否承载、当前运势是否触发。
- 调候用于解释环境与节律需求，不直接等同发财、婚姻或职业吉凶。

## 紫微

主证据顺序：专题宫位本宫 → 对宫与三方四正 → 主星/辅煞结构 → 大限 → 流年 → 流月/流日。空宫要读取借星，但不得把借星当作与本宫坐星完全等价。

- 单纯吉星数量或煞星数量不能决定宫位吉凶；`strength` 只作为筛选信号。
- 禄、权、科、忌要同时看落入哪个流动宫位、对应哪个专题和哪一层时间，不能脱离宫位单独解释。
- 本命宫位强而流年受压，可写“长期能力仍在，但当前阶段成本上升”；流年吉不能覆盖本命结构性限制。
- 出生时辰未知时，不断命宫、身宫、夫妻宫、官禄宫及精细流限。

## 印占

主证据顺序：上升与宫主 → 行星落宫及尊贵状态 → 当前大运/分运 → 相关分盘 → 宫位点数 → 行运。任何单项都不能独立承担完整事件结论。

- 宫位总点反映相关领域的结构承载力，不直接等于事件必成或必败。
- 当前大运和分运主星必须结合其落宫、尊贵、燃烧、逆行及所主宫位解释；只报主星名称不构成断法。
- 关系优先核对本命第七宫及关系分盘；事业优先核对第十宫及事业分盘。分盘出生时间敏感，时辰不准时整体降级。
- 强度精算不可用时，不得用“某星绝对强弱”作核心证据；只能使用已有尊贵度、燃烧、逆行、宫位和点数。

## 西占

主证据顺序：本命行星与轴点 → 宫位及宫主 → 本命相位 → 当前行运 → 需要时再使用更细时间技术。行运必须触发与问题相关的本命点，不能用无关的强相位回答所有专题。

- 相位逼近通常比已经分离更适合描述正在形成的影响；分离相位主要作为余波。
- 合相的性质取决于参与行星和落点，不预设为吉或凶。
- 单条行运只能说明一个触发，不足以预测具体事件。两条以上相关触发、且本命结构允许时，才能提高事件窗口置信度。
- 时辰未知时，不使用上升、天顶、宫位和依赖轴点的判断。

## 融合

- 先检查是否在回答同一领域、同一时间尺度，再比较方向。长期结构与短期触发不同不算真正冲突。
- 四系统不追求票数相等。与问题最相关、字段最完整、时间尺度最匹配的系统承担主证据，其余用于验证或限定。
- 多套低置信信号不能累加成强结论；一条高质量反证不能因多数票被删除。
- 输出中的“明确、倾向、暂不能判断”由证据完整度决定，不由用户期待、聊天长度或模型成本决定。
