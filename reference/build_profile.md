# 建档与资料核对

## 手动建档

用户可以自然输入出生信息。需要识别并确认：

- 姓名：必填。
- 性别：男 / 女。
- 出生日期与时间：尽量精确到分钟；不确定时说明精度。
- 出生地：尽量到城市或区县。
- 关系备注：可选，例如我、对象、孩子、朋友、合伙人。

建档确认分两档：

- 低风险软确认：姓名、性别、公历日期、分钟级时间、城市/区县都清楚，且无材料冲突时，可以直接建档，并提示“我先按这些信息建档；如果有误你直接纠正我”。不要为了形式确认多打一轮。
- 高风险强确认：疑似农历、公历不明、时间不精确、地点过粗、真太阳时可能跨时辰、23:00 换日、材料与本地盘冲突时，必须先让用户确认再建档。

需要强确认时使用：

```text
我识别到：
姓名：[name]
性别：[gender]
出生：[date] [time]
地点：[birth_place_name]
关系备注：[label 或 未填写]

这些信息对吗？
```

八字默认做真太阳时校正。若用户明确要求按钟表时间排盘，则按用户要求处理。若出生时间靠近时辰边界，解读时要温和提醒“这个时间点比较贴近边界，最好再确认一下出生记录”。

## 自动复审

每次建档、重排、修改出生信息或从材料建档后，都必须自动复审，不把复审做成需要用户主动选择的菜单入口。

复审内容：

- 出生日期是否明确为公历；如果疑似农历，必须确认后再排。
- 出生时间精度：精确到分钟 / 只到小时 / 未知。
- 出生地精度：城市 / 区县 / 乡镇；只到城市时记录为近似坐标。
- 真太阳时是否启用，校正后是否跨时辰。
- 是否靠近 23:00 换日、节气/月界、时辰边界。
- 西占上升/MC、印占上升/月宿、紫微命宫是否贴近边界。
- 四系统是否都生成成功，是否有 `degraded=true` 或关键字段为空。
- 若有材料，材料提取字段与本地排盘结果是否一致。

复审结果分为：

- `stable`：正常解读。
- `caution`：相关模块降级，聊天框温和提示。
- `high_risk`：先请用户补充或确认出生信息，不生成强结论。

普通用户只看简短人话提示；debug 或审盘场景才展示详细复审清单。

## 辅助资料

支持用户把星盘、印占 PDF、紫微命盘、星盘截图等资料放进 `materials/`，然后输入 `a` 或直接说明要使用哪些文件。

如果 `materials/` 里有多个文件，先列出文件名，让用户选择要参与建档/复审的文件。用户选多个文件时，逐个提取可用信息，并做一致性检查。

辅助资料只用来识别出生源信息：

- 姓名。
- 性别。
- 出生日期。
- 出生时间。
- 出生地。
- 经纬度。
- 时区。
- 文件类型或盘种标签，例如 Vedic / 紫微 / Western / 八字。
- 材料中写明的上升、月宿、太阳星座、紫微命宫等关键标签，只能作为复审对照项。

不要采用 PDF 或图片里的星体落座、Nakshatra、上升、宫位图、大限、流年边界作为排盘事实。命盘事实只来自本地引擎根据出生时间和地点重新计算的结果。

但材料中的关键标签可以用于交叉复审：

- 印占 PDF 写的月宿 / 上升 / Lahiri 等，与本地印占结果对照。
- 紫微命盘写的命宫主星 / 身宫，与本地紫微结果对照。
- 西占截图写的太阳 / 月亮 / 上升，与本地西占结果对照。
- 八字图写的四柱，与本地八字结果对照。

对照一致时，提升复审信心；对照冲突时，不要自动改盘，先提示用户确认出生时间、地点、时区、历法或材料是否属于同一个人。

不要为了普通 PDF 文本提取而把 PDF 渲染成图片。若 PDF 是扫描件或截图文字识别失败，先进入 OCR 或 AI 视觉读取；仍无法识别时，再请用户补充缺失的出生信息。

从文件中识别到内容后，先给用户确认：

```text
我从资料里识别到：
姓名：[name 或 未识别]
出生：[date] [time]
地点：[place 或 未识别]
经纬度：[lat/lon 或 未识别]

这些信息对吗？
```

如果字段不足，逐项说明缺什么。姓名也是建档必需项；没识别到时要请用户补姓名。

材料复审输出不要给普通用户看原始 OCR 或整段 PDF 文本。只给人话结论：

```text
我把你选的资料也一起核过了：
- 出生时间和地点能对上。
- 印占 PDF 的月宿与本地排盘一致。
- 紫微截图的命宫主星与本地排盘一致。

这张盘可以正常解读。
```

若有冲突：

```text
资料里有一处对不上：PDF 写的上升/命宫与本地按出生资料重排的结果不同。
我不会直接采用 PDF 结果；需要先确认出生时间、地点、时区或这份资料是不是同一个人。
```

## 建档成功后的展示

普通用户不看原始数据、代码字段或内部结果。建档成功后只展示人话摘要：

```text
已完成排盘。

姓名：[name]
出生：[solar_birth]
地点：[birth_place_name]
真太阳时：[已校正/未校正]（如有校正，显示有效排盘时间）
八字：[year]年 [month]月 [day]日 [hour]时
日主：[day_master]（[strength]）
喜用：[yong_shen]

接下来：
① 设为当前对象
② 查看简明命盘解读
③ 深度命盘解读（等同 [详细版/1]）
④ 回到主菜单
```

长结果写入 `outputs/` 下的 Markdown 文档，例如 `outputs/命盘解读_[name_or_timestamp].md`。文档只写给用户看的内容，不写原始数据。

## 结束时的入口

每次建档流程结束，都给用户一个自然的下一步：

- 资料不完整：补充缺失信息 / 换一个文件 / 手动输入 / 回主菜单。
- 临时排盘成功：保存为档案 / 简明命盘解读 / 回主菜单。
- 保存成功：设为当前对象 / 今日算命 / 深度命盘解读（[详细版/1]） / 回主菜单。

## 用户近况

命盘事实和用户近况分开保存。出生信息、排盘结果属于命盘；职业、近期关切、人生事件属于近况。

用户主动提到职业、纠结的事、近期变化时，可以单独记住，只用于建议落地。解读时必须先由盘面确定结论，近况不得进入排盘、系统权重或吉凶判断。
