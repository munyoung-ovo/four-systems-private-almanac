# 建档与资料核对

## 手动建档

用户可以自然输入出生信息。需要识别并确认：

- 姓名：必填。
- 性别：男 / 女。
- 出生日期与时间：尽量精确到分钟；不确定时说明精度。
- 出生地：尽量到城市或区县。
- 关系备注：可选，例如我、对象、孩子、朋友、合伙人。

建档前先让用户确认识别结果：

```text
我识别到：
姓名：[name]
性别：[gender]
出生：[date] [time]
地点：[birth_place_name]
关系备注：[label 或 未填写]

这些信息对吗？
```

八字默认做真太阳时校正。若用户明确要求按钟表时间排盘，则按用户要求处理。若出生时间靠近时辰边界，解读时要温和提醒“这个时间点比较贴近边界，最好再确认一下出生记录”。

## 辅助资料

支持用户把星盘、印占 PDF 等资料放进 `materials/`，然后输入 `a`。

辅助资料只用来识别出生源信息：

- 姓名。
- 性别。
- 出生日期。
- 出生时间。
- 出生地。
- 经纬度。
- 时区。

不要采用 PDF 里的星体落座、Nakshatra、上升、宫位图、大限、流年边界作为排盘事实。命盘事实只来自本地引擎根据出生时间和地点重新计算的结果。

不要把 PDF 渲染成图片。文字识别失败时，直接请用户补充缺失的出生信息。

从文件中识别到内容后，先给用户确认：

```text
我从资料里识别到：
姓名：[name 或 未识别]
出生：[date] [time]
地点：[place 或 未识别]
经纬度：[lat/lon 或 未识别]

这些信息对吗？
```

如果字段不足，逐项说明缺什么。姓名也是建档必需项；没识别到时要请用户补姓名。

## 建档成功后的展示

普通用户不看原始数据、代码字段或内部结果。建档成功后只展示人话摘要：

```text
已完成排盘。

姓名：[name]
出生：[solar_birth]
地点：[birth_place_name]
真太阳时：[已校正/未校正]（如有校正，显示有效排盘时间）
八字：[year]年 [month]月 [day]日 [hour]时
日主：[day_master]（[strength]）
喜用：[yong_shen]

接下来：
① 保存为新档案
② 查看简明命盘解读
③ 回到主菜单
```

长结果写入 `outputs/` 下的 Markdown 文档，例如 `outputs/命盘解读_[name_or_timestamp].md`。文档只写给用户看的内容，不写原始数据。

## 结束时的入口

每次建档流程结束，都给用户一个自然的下一步：

- 资料不完整：补充缺失信息 / 换一个文件 / 手动输入 / 回主菜单。
- 临时排盘成功：保存为档案 / 简明命盘解读 / 回主菜单。
- 保存成功：设为当前对象 / 今日算命 / 深度命盘解读 / 回主菜单。

## 用户近况

命盘事实和用户近况分开保存。出生信息、排盘结果属于命盘；职业、近期关切、人生事件属于近况。

用户主动提到职业、纠结的事、近期变化时，可以记住，用于之后的今日建议和深度解读。
