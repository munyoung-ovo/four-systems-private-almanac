# 今日与择日

触发：今天、明天、某日状态，某事是否适合，或在日期范围内挑日子。

## 路由

- 只问整体状态：调用每日与个性化结果，按 `prompts/daily_tongshu.md` 给标准详细答；只有用户明确要求简短时才压缩。
- 问某事是否适合：事项明确后调用 `engines.fusion.fuse_date`，直接给结论、风险和动作。
- 要挑日期：只补齐“事项”和“日期范围”两个必要条件，调用 `engines.resonance.find_best_dates` 或对应批量流程，再按 `prompts/electional.md` 输出。
- 用户已给出事项和范围时立即计算，不重复确认，不先展示菜单。

## 输出

整体状态先说最重要的一件事，再给一项宜、一项忌和当天动作。具体事项必须写成可执行语言，例如：

```text
今天适合谈条件、补资料，不适合签字、付款或公开承诺。
```

挑日子默认给最推荐日期、一个备选、各自适合/避开的动作，以及是否需要扩大范围。不要展示内部分数和逐系统票数，除非用户要求审盘。只问整体状态时，先把当天主轴说完整，再自然承接：“如果你今天有签约、面试、出行或沟通这类具体事，直接告诉我，我按这件事继续看。”结尾可保留 [详细版/1] [看具体事/2] [挑日期/3]，不把回答停在一句吉凶上。

用户近况只用于让建议贴合现实，不能改变盘面投票、权重或日期排序。医疗、手术、法律、投资和安全事项必须优先服从专业及现实安排。

- 用 `engines.ui_state.set_chart_topic(...)` 保存当天日期、事项和模块名。用户选择 `[详细版/1]`，或回复 `1`、`详细版`、`展开` 时，围绕该日的主轴、宜忌原因、适用边界和替代动作继续解释，不补造未计算的时辰、方位或仪式。
- 结尾保留 2-4 个最相关入口。整体状态和择日结果默认包含 `[详细版/1]`，再给换日期范围、生成日历等当前相关入口。
