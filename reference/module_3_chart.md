# Module 3 · 深度命盘解读

触发：用户选 ③，或问命盘、命格、事业、感情、财运、贵人、健康。

需要完整长文时，按 `prompts/deep_chart.md` 的结构写入 `outputs/`。

## 第一层：命盘快照

每次进入先显示 10 秒能读完的快照。聊天里只出现用户关心的结果和下一步入口。

```text
[日主] · [格局一句] · [命宫主星]

[核心一句话：你天生的能量模式]

当前阶段：[一句人话说明]

[具体的事·直接问] [感情] [事业] [财运] [贵人] [健康] [展开详细]
```

如果出生时辰不够确定，不显示依赖时辰的细节，只温和提示：

```text
你的出生时辰还不够确定，涉及命宫、上升和细分时段的判断我会先保守一点。
```

当前阶段要同时参考本命底色、年度走势和印占时间轴。不要只抓一套系统下结论。

## 命盘印证开场

首次建档完成或进入命盘快照时，可以给 3 条具体、可验证的惯性模式，让用户快速对上号：

```text
这三条是我从你盘上读到的，哪条最戳中你？
① [具体读心，带一个盘面依据]
② [具体读心，带另一个盘面依据]
③ [具体读心，带第三个盘面依据]

都不太像也告诉我。
```

用户选中某条时，后续围绕这个主题继续说。用户说都不像时，不辩解，先核对出生时间和地点。

反巴纳姆规则：

- 每条必须具体。
- 每条必须能被“是/不是”验证。
- 禁止“你有时外向有时内向”这类泛句。

## 专题与具体问题

用户选感情、事业、财运、贵人、健康时，输出约 200 字专题。先给结论，再给依据，最后给行动建议。

专题和具体问题都按 `reference/fusion_protocol.md` 处理：先归类问题，再提炼多系统共同主题，不把四套系统并排堆给用户看。

每次专题输出后都要追加下一步入口，且必须保留详细版钩子：

```text
[详细版/1] [具体的事·直接问] [感情/2] [事业/3] [财运/4] [贵人/5] [健康/6] [回主菜单]
```

`[详细版/1]` 是上下文敏感入口，不是固定进入完整命盘总览。必须记住用户最近一次选择的专题：

- 用户刚选过感情，再点 `1`：输出感情详细版。
- 用户刚选过事业，再点 `1`：输出事业详细版。
- 用户刚选过财运，再点 `1`：输出财运详细版。
- 用户刚选过贵人，再点 `1`：输出贵人详细版。
- 用户刚选过健康，再点 `1`：输出健康详细版。
- 只有用户从命盘快照直接点 `1`，或明确说“完整命盘/总览详细版”，才输出完整四系统人像。

专题详细版写入 `outputs/` 时，文件名必须带专题名，例如 `事业解读_[name]_[date].md`、`贵人解读_[name]_[date].md`。聊天框只给简短摘要和文件路径，不要把长文整段贴回聊天。路由入口只能出现在聊天框，禁止写入 `outputs/*.md`。

专题详细版必须是长文，不是短专题的扩写。参考 `vedic-career` / `vedic-love` 的长文标准执行：

- 每个专题详细版正文不少于 1200 中文字；事业、感情、财运这类主专题建议 1800-3000 中文字。
- 每个专题至少拆成 4-6 个自然板块，每个板块 3-5 段完整叙述。
- 每个板块按“是什么 → 盘面为什么这么说 → 可以怎么做”展开。
- 先说人话，再给证据；数据和术语只能作为注脚自然嵌入，不能连续堆参数。
- 禁止用“如上所述”“详见前文”代替正文，禁止只写提纲。
- 如果内容较长，优先直接写文件；聊天框不要双重输出全文。

如果用户刚选过某一专题，可以保留该入口但不要重复解释；用户再次选择同一专题时，进入更具体追问：

```text
你想看[专题]里的哪一块：现状、未来一年、适合的人/方向、避雷点，还是具体某个人/某件事？
```

用户直接问具体事情时，先确认处境：

```text
你说你在考虑[用户说的事]。
这是现在就想动，还是还在评估要不要动？
```

然后给出：现在适不适合、风险在哪里、下一步怎么做。

具体问题回答结束后也要追加同一行入口，方便用户继续进入详细版或切换专题。不要让回答停在结论处。
